HTML5 no es un nuevo lenguaje. Es una evolución más que revolución.

HTML

Tim Berners-Lee se basó sobre SGML (Standard Generalized Markup Language) para escribir 12 etiquetas HTML para crear páginas web. El uso de etiquetas no es de él, lo que hizo fue utilizar algo que ya existía para crear otra cosa.

1991

HTML4

HTML1 nunca existió, la primer versión fue HTML2, publicada por IEFT (Internet Engineering Task Force) y constituída mayormente por implementaciones existentes. IEFT fue reemplazado por W3C (World Wide Web Consortium) www.w3.org que en 1999 publicó HTML 4.01

Después de HTML 4.01 la siguiente versión fue nombrada XHTML 1.0 y no fueron agredados nuevos elementos, sólo se modificó la sintaxis.

XHTML requería a los autores seguir las reglas de XML (Extensible Markup Language). Lenguaje de marcas desarrollado por el W3C utilizado para almacenar datos en forma legible. Deriva del SGML.

Coincide con XHTML la aparición y soporte de los navegadores web de CSS 1.0.

Al poco tiempo W3C publicó XHTML 1.1 que se centraba aún más en XML.

La versión XHTML 2 provocó malestar general ya que no contemplaba compatibilidad con contenidos anteriores.

Opera, Apple y Mozilla se unieron en contra de W3C y formaron WHATWG (Web Hypertext Application Technology Working Group).

1999

HTML5

WHATWG estaba dividido en 2 especificaciones: Web Forms y Web Apps, se unifican en una sola especificación llamada simplemente HTML5.

2005

WHATWG

W3C decide usar las bases de HTML de WHATWG. Simultáneamente también seguía trabajando con XHTML 2.

2006

No más XHTML 2

W3C anuncia que no será actualizado más XHTML 2.

En ese entonces había 2 grupos trabajando con HTML5, W3C y WHATWG.

WHATWG publicaba, luego revisaba. W3C revisaba, luego publicaba.

2009

Recomendación

HTML5 se puso ofialmente como uso recomendado.

HTML5 Oficialmente será estandarizado en el 2022.

Lo que importa realmente es cuando los navegadores web comienzan a soportarlo.
